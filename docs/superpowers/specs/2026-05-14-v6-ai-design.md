## 一、背景与动机

### 1.1 现状问题

v6.0-beta 自研的 AI 分析模块（`ai_analyzer.py`）存在根本性缺陷：
- 用户需要自己配置 API Key，门槛高
- 只能做简单的 JSON 结构化分析，能力有限
- 没有对话交互，用户无法追问
- LLM 调用是单次的，无法做多轮推理
- OCR、去重、分析三个模块各自独立，无法协同

### 1.2 核心思路

**用 [Claude Code Haha](https://github.com/NanmiCoder/cc-haha) 替代自研 AI 模块。**

Claude Code 是一个完整的 AI 编程助手，具备：
- 多模型支持（Anthropic、OpenAI、本地模型）
- 工具调用能力（文件读写、终端执行、浏览器控制）
- Computer Use（截图、点击、输入）
- Skills 系统（可扩展能力插件）
- 记忆系统（跨会话持久化）
- 多 Agent 编排（并行任务执行）

把这些能力用在 CIE 试卷场景，远比自研的 `analyze_paper(text) -> JSON` 强大得多。

### 2.2 关键设计决策

| 决策 | 选择 | 理由 |
|---|---|---|
| 集成方式 | CLI 子进程 + IPC | cc-haha 本身是 CLI 工具，通过 stdin/stdout JSON 通信 |
| Skills | 自定义 CIE Skills | 利用 cc-haha 的 Skills 系统，写 CIE 领域专用 skill |
| 本地 vs 云端 | 混合 | Claude Code 引擎本地运行，模型调用走云端 API |
| OCR | 保留本地 Tesseract | OCR 是纯本地图像处理，不需要 LLM |
| 去重 | 保留本地 embedding | 向量计算是本地的，但可以用 AI 辅助判断语义相似 |

## 四、技术实现

### 4.1 Claude Code CLI 集成

**方式：子进程 + JSON IPC**

```python
# src/backend/claude_engine.py

import subprocess
import json
import threading

class ClaudeEngine:
    """Claude Code Haha 引擎封装"""

    def __init__(self, cli_path="claude-haha"):
        self._cli_path = cli_path
        self._process = None
        self._callbacks = {}

    def start(self, work_dir=None):
        """启动 Claude Code CLI 子进程"""
        cmd = [self._cli_path, "--output-format", "json"]
        self._process = subprocess.Popen(
            cmd,
            stdin=subprocess.PIPE,
            stdout=subprocess.PIPE,
            stderr=subprocess.PIPE,
            cwd=work_dir,
            text=True,
            bufsize=1,
        )
        # 启动读取线程
        threading.Thread(target=self._read_loop, daemon=True).start()

    def send_message(self, message, callback):
        """发送消息到 Claude Code"""
        request = {"type": "message", "content": message}
        self._process.stdin.write(json.dumps(request) + "\n")
        self._process.stdin.flush()
        self._callbacks[request["type"]] = callback

    def _read_loop(self):
        """持续读取 Claude Code 输出"""
        for line in self._process.stdout:
            try:
                data = json.loads(line.strip())
                self._handle_response(data)
            except json.JSONDecodeError:
                continue
```

### 4.4 API Bridge 层

```python
# src/backend/api.py 新增方法

def ai_start(self):
    """启动 Claude Code 引擎"""
    self._claude = ClaudeEngine()
    self._claude.start(work_dir=os.path.expanduser("~"))
    return json.dumps({"ok": True})

def ai_send(self, message, context=""):
    """发送消息到 AI 助手"""
    if context:
        full_msg = f"[上下文: {context}]\n\n{message}"
    else:
        full_msg = message
    self._claude.send_message(full_msg, callback=self._on_ai_response)
    return json.dumps({"ok": True})

def ai_get_response(self):
    """获取 AI 最新回复（轮询）"""
    return json.dumps({
        "ok": True,
        "messages": self._ai_messages[-50:]  # 最近 50 条
    })

def ai_analyze_paper(self, pdf_path):
    """分析试卷（快捷方法）"""
    ocr_result = self._ocr.extract_text(pdf_path)
    context = f"试卷文本:\n{ocr_result.full_text[:8000]}"
    return self.ai_send("请分析这份试卷的考点分布、难度和重复题型", context)

def ai_solve_question(self, question_text):
    """请求解题（快捷方法）"""
    return self.ai_send(f"请解答这道题并给出详细步骤:\n{question_text}")

def ai_study_plan(self):
    """生成学习计划"""
    # 获取用户下载的试卷列表
    downloads = self.get_download_history()
    return self.ai_send(
        "请基于我下载的试卷生成一份复习计划",
        context=f"我的试卷: {downloads}"
    )
```

### 4.5 前端 JS

```javascript
// AI 助手面板逻辑

let aiMessages = [];

async function aiSendMessage() {
    const input = document.getElementById('ai-input');
    const msg = input.value.trim();
    if (!msg) return;

    // 显示用户消息
    aiMessages.push({role: 'user', content: msg});
    renderAIMessages();
    input.value = '';

    // 发送到后端
    await pywebview.api.ai_send(msg);

    // 轮询回复
    pollAIResponse();
}

async function pollAIResponse() {
    const interval = setInterval(async () => {
        const r = JSON.parse(await pywebview.api.ai_get_response());
        if (r.messages.length > aiMessages.length) {
            aiMessages = r.messages;
            renderAIMessages();
            // 检查是否完成
            const last = r.messages[r.messages.length - 1];
            if (last && last.role === 'assistant' && last.complete) {
                clearInterval(interval);
            }
        }
    }, 500);
}

function renderAIMessages() {
    const el = document.getElementById('ai-messages');
    el.innerHTML = aiMessages.map(m =>
        `<div class="ai-msg ai-${m.role}">
            <div class="ai-avatar">${m.role === 'user' ? '👤' : '🤖'}</div>
            <div class="ai-content">${marked(m.content)}</div>
        </div>`
    ).join('');
    el.scrollTop = el.scrollHeight;
}

// 快捷操作
async function aiAnalyzeCurrent() {
    const selected = S.lastResults?.[0];

    const pdfPath = S.downloadDir + '/' + selected.filename;
    await pywebview.api.ai_analyze_paper(pdfPath);
    pollAIResponse();
```

## 五、与现有模块的关系

### 5.1 保留的模块

| 模块 | 保留理由 |
|---|---|
| `ocr_engine.py` | OCR 是本地图像处理，不需要 LLM。Claude Code 读取 OCR 结果 |
| `dedup_engine.py` | 向量计算是本地的，但 Claude Code 可以辅助判断语义 |
| `fts_engine.py` | 全文搜索是本地 SQLite，不需要 AI |

### 5.2 替换的模块

| 原模块 | 替换为 |
|---|---|
| `ai_analyzer.py` | `claude_engine.py`（Claude Code CLI 封装） |
| LLM API 配置界面 | Claude Code 自带的提供商配置 |
| 单次分析 → JSON | 多轮对话 + Skills |

### 5.3 删除的代码

- `ai_analyzer.py` 中的 `LLMProvider`、`OpenAIProvider`、`AnthropicProvider`
- `api.py` 中的 `configure_ai`、`get_ai_config`、`test_ai_connection`
- `ui_v2.html` 中的 AI 配置表单（provider/key/model）
- `requirements.txt` 中的 `openai`、`anthropic`

## 八、风险与缓解

| 风险 | 缓解 |
|---|---|
| cc-haha 安装复杂 | 提供一键安装脚本，或打包到 C-Paper 安装包 |
| CLI 子进程不稳定 | 实现崩溃检测和自动重启 |
| 模型 API 费用 | cc-haha 支持免费/低成本模型（如 DeepSeek、Ollama） |
| 多轮对话占用上下文窗口 | 自动摘要旧对话，保留关键信息 |
| Skills 维护成本 | Skills 是 Markdown 文件，社区可贡献 |
